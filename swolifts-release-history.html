SwoLifts

Release History

Version notes for SwoLifts features, fixes, and product decisions.

Versioning Model

How SwoLifts versions are tracked

  • Major versions mark large product shifts, major workflow changes, or compatibility-breaking changes.
  • Minor versions add user-facing features or meaningful UX improvements while preserving existing data.
  • Patch versions cover bug fixes, performance improvements, copy changes, and low-risk polish.
Current Version

SwoLifts 1.0

SwoLifts 1.0 is the current development baseline for the planned first public release.

Planned launch baseline

Workout Tracking

  • Active workout logging with autosave.
  • Exercise templates for repeat routines.
  • Rest timer and persistent workout notification.
  • Set timestamps captured when sets become complete.

Exercise Data

  • Preloaded exercise library with custom exercises.
  • External weight, bodyweight, weighted bodyweight, and assisted bodyweight load types.
  • Assistance weight support for assisted pull-ups, assisted dips, and similar movements.
  • RIR tracking option for effort-aware strength estimates.

Progress and Ownership

  • Personal records, exercise trends, weekly volume, muscle volume, and bodyweight charts.
  • Plate Calculator for bar loading.
  • Workout history with editable completed sessions.
  • Offline local storage with ZIP export/import backup.
Release Note Format

How future entries will be documented

This section is a reminder for how future SwoLifts releases should be written once new versions are published.

  • Version number such as 1.0.1, 1.1.0, or 2.0.0.
  • Release date once the update is published.
  • New features added in that version.
  • Fixes and improvements that changed behavior, performance, or usability.
  • Data migration notes when a release changes local storage or import/export behavior.
  • Known limitations when something needs follow-up or further discussion.
